Overview

This is the main class you use to interface with the Upsight framework. It has a number of class methods you use to send events, get status information or display content.

For more details about adding the framework to your application, see the integration instructions.

+ recordAnalyticsEventWithName:properties:

Sends information to the Upsight server representing events that happen within your application. An event could be anything that is of some interest in your application, e.g. when a user clicks a specific button, adds a friend, etc.

+ (void)recordAnalyticsEventWithName:(NSString *)name properties:(NSDictionary *)properties

Parameters

name

User-defined event name. You are free to assign any name to events to logically group them together. For example you can create events with “monetization” name to group together events related to monetization. Note that . is a special symbol that can be used within event name to create a compound name, where every . represents one level of hierarchy. For example, my_store.weapons and my_store.armor allows you to get metrics on my_store as well as my_store.weapons and my_store.armor using the Upsight Dashboard.
properties

A dictionary containing any information you would like to associate with this event.

+ setUserID:preserve:

Sets the specified User ID. This has several effects: - Ends the session for any current user - Creates a new user if needed (see retainAttributes flag) - Begins a new Session - Once a User ID is set, it is retained through application starts.

+ (nullable NSError *)setUserID:(nonnull NSString *)aUserID preserve:(BOOL)retainAttributes

Parameters

aUserID

A new value for the user attribute indicated by aUserID. It is highly recommended to hash the userID to prevent sending private and indientifiable information. The string must be a maximum of 256 bytes in size.
retainAttributes

a flag to determine if the user attributes from an anonymous user session should be copied into a new user. If transitioning to or from an existing user, this flag is ignored.
